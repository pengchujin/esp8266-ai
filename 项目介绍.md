# AI Clock 项目介绍

## 项目概述

AI Clock 是一个桌面状态时钟，通过 ESP8266 驱动 1.54 寸 TFT 屏幕，**默认显示天气时钟页面**。当检测到 Claude Code / Codex CLI 正在工作时自动切换到桌宠动画页面，工作结束后恢复时钟页面。配合 Mac/Windows 桥接程序使用，支持 WiFi 或 USB 串口两种连接方式，是 AI 开发者的桌面伴侣。

## 硬件系统

### 核心芯片：ESP8266（ESP-12S 模组）

| 参数 | 数值 |
|------|------|
| CPU | Tensilica Xtensa LX106 @ 80MHz |
| RAM | **80KB**（其中约 50KB 可供用户程序使用） |
| Flash | **4MB**（通过 SPI 外挂，存放固件和文件系统） |
| WiFi | IEEE 802.11 b/g/n 2.4GHz |
| GPIO | 11 个可用引脚 |
| 供电 | 5V Micro-USB |

### RAM 和 Flash 的作用

**RAM（80KB 动态内存）**：
- 存放程序运行时变量、JSON 解析缓冲区、TFT 帧缓冲
- 本项目占用约 61%（~50KB），余量留给 WiFi 协议栈和动态分配
- 受限的 RAM 是设备上解码 GIF 的最大瓶颈（AnimatedGIF 需一次性分配约 24KB 连续内存）

**Flash（4MB 闪存）**：
- 存放固件程序（编译后约 875KB）
- **LittleFS 文件系统**（占用剩余空间）：
  - 自定义桌宠精灵（上传的 GIF 解码后存储于此）
  - 壁纸背景缓存（240×240 RGB565 位图 = 115KB）
  - 亮度设置、桥接主机地址等配置持久化
- Flash 是**非易失性**存储，断电后数据不丢失

### 显示面板：ST7789 TFT

| 参数 | 数值 |
|------|------|
| 尺寸 | 1.54 英寸 |
| 分辨率 | 240 × 240 像素 |
| 接口 | **SPI**（4 线串行外设接口） |
| 色彩 | RGB565（65K 色，每像素 2 字节） |
| 背光 | PWM 可调（GPIO5 控制，0~100 级亮度） |

**引脚映射**（遵循 SD2/小电视 硬件设计）：

| 功能 | GPIO | 说明 |
|------|------|------|
| MOSI | D7 (GPIO13) | SPI 数据发送 |
| SCLK | D5 (GPIO14) | SPI 时钟信号 |
| CS | D8 (GPIO15) | SPI 片选 |
| DC | D3 (GPIO0) | 数据/命令切换 |
| RST | D4 (GPIO2) | 复位 |
| BL | D1 (GPIO5) | 背光（低电平激活，PWM 调节） |

### CH340 USB 转串口芯片

**CH340 是什么：**
CH340 是一款由中国南京沁恒微电子（WCH）设计的 **USB 转 UART 串口芯片**。它在一端连接电脑 USB 口，另一端连接 ESP8266 的 TX/RX 引脚，让电脑能通过虚拟串口与 ESP8266 通信。

**CH340 在项目中的作用：**

| 功能 | 说明 |
|------|------|
| **烧录固件** | PC 通过 CH340 发送编译好的 .bin 文件到 ESP8266 的 Flash，工具链为 `esptool.py` |
| **串口通信** | Mac 端的 AIClockBridge 通过 CH340 虚拟串口（`/dev/cu.usbserial-130`）向设备推送状态数据（#STATUS、#NET 等帧），实现无 WiFi 运行 |
| **调试日志** | ESP8266 的 `Serial.print()` 输出经 CH340 传回电脑，可在串口监视器查看 |
| **自动复位** | CH340 的 DTR/RTS 引脚通过自动复位电路连接 ESP8266 的 RST/GPIO0，刷机时自动进入下载模式，无需手动按键 |

**CH340 驱动的虚拟串口**：
- macOS：`/dev/cu.usbserial-130`
- Windows：`COM3`（示例）
- 波特率：**115200 bps**

## 软件系统

### 固件（ESP8266 端）

| 项目 | 说明 |
|------|------|
| **开发框架** | Arduino（C++） |
| **构建工具** | PlatformIO |
| **固件版本** | v0.6.0 |
| **代码量** | 约 3100 行 C++（`src/main.cpp`） |
| **目标板** | `nodemcuv2`（兼容 ESP-12E/ESP-12S） |

**主要依赖库**：

| 库 | 版本 | 用途 |
|----|------|------|
| TFT_eSPI | ^2.5.43 | ST7789 显示屏驱动 |
| WiFiManager | ^2.0.17 | WiFi 配网（捕获门户 AP 模式） |
| ArduinoJson | ^7.2.1 | JSON 解析（状态、天气、股票等数据） |
| AnimatedGIF | ^2.2.0 | 用户上传的 GIF 动图解码为精灵帧 |
| LittleFS | 内置 | Flash 文件系统 |

**系统架构**：

```
setup()           → 初始化 LittleFS → 加载配置 → WiFi 连网
                     → 启动 Web 服务器 → NTP 对时 → 显示主界面

loop()            → WiFiManager.process()     （WiFi 配网门户维护）
                     → pumpSerial()            （USB 串口帧接收）
                     → webServer.handleClient()（HTTP 请求处理）
                     → pollBridge()            （WiFi 轮询桥接程序）
                     → tick / redraw           （显示刷新）
```

**显示模式**（7 种页面）：

| 模式 | 命令 | 说明 |
|------|------|------|
| **CLOCK** | `mode=clock` | 🕐 **天气时钟页（开机默认）** |
| CLAUDE | `mode=claude` | 固定显示 Claude 桌宠 + 配额环 |
| CODEX | `mode=codex` | 固定显示 Codex 桌宠 + 配额环 |
| AUTO | `mode=auto` | 根据工作状态自动切换 Claude/Codex |
| NET | `mode=net` | Mac 网速实时波形图（带 CPU/内存） |
| MUSIC | `mode=music` | 当前播放音乐封面 + 进度条 |
| STOCK | `mode=stock` | 自选股行情（最多 4 只） |

**默认行为（v0.6.0）**：

```
开机 → 天气时钟页（默认 CLOCK 模式）
   ↓
检测到 Claude / Codex 正在工作 → 自动切换到桌宠页（牛牛动画 + 配额环）
   ↓
工作结束（空闲）→ 自动回到天气时钟页
   ↓
检测到音乐播放（仅 WiFi 模式）→ 自动切换到音乐页
```

> 切换到桌宠页时，设备会在 Claude 和 Codex 之间切换显示（双方都工作时 2 秒切换，空闲时 6 秒切换）。工作状态通过 Mac 桥接程序的 `/status` 端点判断。**WiFi 模式和 USB 串口模式均支持此自动切换逻辑。**

**HTTP API 端点**（设备作为服务器，监听 80 端口）：

```
GET  /                   → 管理设置网页
GET  /api/info           → 设备信息（IP、模式、亮度、精灵版本等）
POST /api/display        → 切换显示模式
POST /api/bridge         → 设置桥接主机地址
POST /api/brightness     → 调节背光亮度（0-100）
POST /api/wallpaper      → 开关桌面壁纸背景
POST /sprite/claude      → 上传 Claude 角色自定义 GIF
POST /sprite/codex       → 上传 Codex 角色自定义 GIF
POST /sprite/{slot}/reset → 恢复角色默认动画
```

## 通信协议

### WiFi 模式（HTTP）

```
┌──────────┐  每隔 5s GET /status    ┌──────────────┐
│ ESP8266  │ ◄────────────────────── │  AIClockBridge │
│ (设备)   │ ──────────────────────► │  (Mac/PC)     │
└──────────┘  返回 JSON 状态数据      └──────────────┘
     │                                       │
     │  POST /api/display (切换模式)          │
     │ ◄─────────────────────────────────────│
     │                                       │
     │  POST /sprite/claude (上传 GIF)       │
     │ ◄─────────────────────────────────────│
```

### 串口模式（USB）

当设备通过 USB 连接且 WiFi 不可用时，走串口帧协议（115200 bps）：

**桥接 → 设备（下行）**：

```
#HELLO                          → 握手
#STATUS {"claude":...}          → Claude/Codex 状态 JSON
#NET {"rx_bps":...}             → 网速数据 JSON
#STOCK {"stocks":[...]}         → 股票数据 JSON
#WEATHER {"temp":29.1,...}      → 天气数据 JSON
#DEEPSEEK {"bal":41.25,...}     → DeepSeek 余额 JSON
#MUSIC {"title":"..."}          → 音乐信息 JSON
#BG <rev> <b64len>              → 壁纸批量推送头
#STRIP <id> <rev> <b64len>      → CJK 文字条批量推送头
#C <base64 chunk>               → 批量推送数据块
#E                              → 批量推送结束标记
```

**设备 → 桥接（上行）**：

```
#DEVICE {"name":"aiclock","fw":"0.6.0"}  → 设备身份声明
#NEED                                    → 请求重新推送
```

串口帧是一行一个，以换行符分隔。每种帧以 `#` 开头，桥接程序据此区分数据帧和设备日志。

## 功能页面详解

### 时钟天气页（MODE_CLOCK）

```
┌──────────────────────────────┐
│ 上海·多云     ☀️ 天气图标     │  ← CJK 城市名 + WMO 天气码图标
│ 最高33° 最低26°              │  ← 温区文字条
│                              │
│        12:34:56              │  ← 白色 HH:MM（冒号闪）+ 橙色 MM + 小秒
│      7月19日 周六             │  ← CJK 日期条
│                              │
│ 🌡 ═══════ 29°  温度条(红)      │  ← 数据行：图标 + 颜色条 + 数值
│ 💧 ═══════ 85%  湿度条(绿)      │
│                              │
│ ● DPS CNY  41.25  余额      │  ← DeepSeek 余额（蓝灰色 DPS 标签）
│                         🐮   │  ← 右下角精灵（原作牛牛）
└──────────────────────────────┘
```

- **背景**：合成 Mac 桌面壁纸（55% 遮罩变暗 + CJK 文字叠加），或纯黑（壁纸关闭时）
- **天气数据源**：Open-Meteo API（免费，无需 API Key，每 10 分钟刷新）
- **NTP 对时**：`ntp.aliyun.com`，时区 CST-8
- **角标精灵**：右下角 80×80 区域显示当前 App（Claude 或 Codex）的原作牛牛精灵动画，支持自定义上传的 GIF；时钟页与桌宠页共享同一套动画数据
- **自动切换**：当 Claude 或 Codex 正在工作时，屏幕自动从时钟页切换到桌宠页显示工作状态和配额环；工作空闲后自动恢复时钟页。WiFi 和 USB 串口模式均支持此逻辑

### 桌宠页（Claude/Codex 自动切换）

```
┌──────────────────────────────┐
│    ┌──────────────────┐      │
│    │                  │      │  ← 绿色配额环（工作时）/ 红色闪烁（桥断）
│    │    🐮 牛牛动画    │      │  ← Claude 111×120 / Codex 120×120 精灵
│    │                  │      │
│    └──────────────────┘      │
│        5h: 42%  Wk: 67%      │  ← 配额百分比（5 小时 / 7 天）
│                              │
│        ⌂ Claude logo         │  ← 左上角 App 图标
└──────────────────────────────┘
```

- **配额环**：外圈进度条表示当前配额用量，绿色 = 正常，红色闪烁 = 桥接断开需检查
- **桌宠动画**：工作态循环播放牛牛行走动画（8.3 fps）；闲置态显示静止帧
- **自定义精灵**：可通过 Mac 应用的「更换桌宠动画」功能上传 petdex.dev 的 GIF，设备端解码后立刻替换；恢复默认即可回到原作牛牛
